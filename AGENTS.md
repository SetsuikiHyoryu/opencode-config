## Markdown 书写规则

1. 写完或改完 Markdown 文件后，  
   必须校验一遍是否符合 [markdownlint][markdownlint] 规则，  
   可以仅逐条肉眼核对，但这一步不能省略。
   - 表格行遭遇 MD013 时优先在保证不丢失的前提下简化内容，  
     如果简化不了，应采取列表、小标题及段落等其他方式，  
     如果必须保持表格形式，则允许用 `<br>` 让渲染结果表现为换行。  
     如果吃不准，需询问用户。
2. 句子使用句号及其他表示终止的符号结尾。
   - 标题中不能有标点符号。
   - 可以有说明用的括号、横线等，不明确时询问用户。
3. 换行时如果需要真实渲染为两行，必须在行尾加上双尾空格。
   - 处理 MD013 时应视为真实渲染，尽量在句号处换行，  
     没有句号时尽可能在标点符号处换行。
4. 并列的清单/枚举类内容，直接写成列表，不要靠标点把多个并列项硬塞进一整句话。

[markdownlint]: https://github.com/DavidAnson/markdownlint#rules--aliases
