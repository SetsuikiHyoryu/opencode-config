## Markdown 书写规则

1. 每份 Markdown 文件的最终输出完成前，  
   必须校验一遍是否符合 [markdownlint][markdownlint] 规则，  
   可以仅逐条肉眼核对，但这一步不能省略。
2. 每份 Markdown 文件的最终输出完成前，逐行检查行长。
   - 超过 80 字符的行在最近的标点处断行，断行前的行尾加双尾空格。
   - 表格行超过 80 字符时优先在保证不丢失的前提下简化内容。
     - 如果简化不了，应将表格改为列表、小标题或段落等其他方式。
     - 如果为了展示效果必须保持表格形式，则需要使用 `<br>` 让渲染结果表现为换行。  
       此时忽略 MD013, MD033 没有关系。
     - 如果有任何不确定的地方，询问用户。
3. 每份 Markdown 文件的最终输出完成前，逐行检查两行之间是否有空行。  
   没有空行且不是列表元素时，两行之中前一行的行尾加双尾空格。
4. 句子使用句号及其他表示终止的符号结尾。
   - 标题中不能有标点符号。
   - 可以有说明用的括号、横线等，不明确时询问用户。
5. 并列的清单/枚举类内容，直接写成列表，不要靠标点把多个并列项硬塞进一整句话。

[markdownlint]: https://github.com/DavidAnson/markdownlint#rules--aliases
